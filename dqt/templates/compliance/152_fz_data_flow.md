# 152-ФЗ — поток персональных данных в DQT

> Документ описывает контролы, которые DQT обеспечивает оператору
> данных. Конкретные обязательства (уведомление Роскомнадзора, согласие
> субъекта, политика обработки ПДн) лежат на стороне оператора —
> DQT даёт инструменты для compliance-отчётности, а не сами решения.

## 1. Где данные находятся в DQT

| Слой | Хранение | Управление |
|---|---|---|
| Загрузка через UI | base64 в HTTP-теле, парсинг в pandas | в памяти worker'а; cap `DQT_MAX_UPLOAD_MB` |
| Сессии | RAM по умолчанию; на диске при `DQT_SESSION_DIR` (parquet + json) | TTL `4 ч`, sweeper удаляет; шифрование через `DQT_ENCRYPTION_KEY` |
| Runs DB | SQLite `~/.dqt/runs.db` или Postgres (`DQT_RUNS_DSN`) | агрегаты + verdict; **без сырых строк ПДн** |
| Baselines | parquet в `DQT_BASELINES_DIR` | оператор контролирует доступ к директории |
| Audit log | JSONL в `DQT_AUDIT_LOG`, опционально с HMAC | tamper-evident, легко отдать комплаенсу |
| HTML отчёт | один файл, эмбедные Plotly | оператор хранит; обычно short-lived |

DQT **не делает** исходящих запросов к третьим сторонам по умолчанию.
Webhook-уведомления (Slack/Teams/собственный URL) активируются явно
оператором.

## 2. Категории данных

DQT работает с табличными данными, которые **сам по себе классифицирует
как обезличенные** (имена признаков, агрегаты по бакетам). Однако
загруженные строки могут содержать ПДн (ФИО, паспорт, телефон) — в этом
случае ответственность за защиту ложится на оператора.

Рекомендации:

* загружать **уже обезличенные** датасеты (хеш ID, обезличенные
  адреса, replay-мосты ID хранятся отдельно);
* drill-down samples (Python API) хранятся только в памяти; для
  `?session=<sid>` URL они недоступны после рестарта без
  `DQT_SESSION_DIR`;
* для on-prem банка PSI и stability считаются на агрегатах — в
  audit log выгружается имя признака и числовое значение PSI, но не
  raw-выборка.

## 3. Срок хранения

| Артефакт | По умолчанию | Где переопределить |
|---|---|---|
| Сессии | 4 ч TTL | `SessionStore(ttl_seconds=...)` |
| Runs | бессрочно | удаление через `dqt runs delete <id>` |
| Baselines | бессрочно | удаление через `dqt baseline delete <name>` |
| Audit log | append-only | внешняя ротация (logrotate) |

Удалите Runs / Baselines, относящиеся к прекратившим действие
обязательствам обработки. DQT не стирает за вас — это сознательный
выбор: команда compliance должна явно подтверждать удаление.

## 4. Канал связи

* HTTP-сервер по умолчанию слушает `127.0.0.1:8050`. Для внешнего
  доступа разворачивается за nginx + Let's Encrypt
  (см. `deploy/install.sh`).
* Все вебхуки идут поверх HTTPS (если URL начинается с `https://`).
* Никакого outbound-трафика на сервисы Anthropic / OpenAI / Google /
  AWS из DQT.

## 5. Удаление данных

* `dqt runs delete <id>` — удаляет run, его offenders и связанные
  performance-метрики.
* `STORE.reset(sid)` — удаляет сессию и parquet/json sidecar.
* `dqt baseline delete <name>` — удаляет baseline и его parquet.

В Postgres-режиме (`DQT_RUNS_DSN`) команда удаления выполняется
на стороне БД (CASCADE прописывать дополнительно при необходимости).

## 6. Аудит

При установленной `DQT_AUDIT_HMAC_KEY` каждая запись audit log
подписана HMAC-SHA256, образуя цепочку: подделка любой исторической
строки ломает все последующие подписи. Это не блокчейн, а
tamper-evident контроль, достаточный для проверки регулятором.

## 7. Air-gapped развёртывание

```bash
# 1. Зеркало образа в корпоративный registry
docker pull ghcr.io/gorevds/dqt-ui:1.2
docker tag ghcr.io/gorevds/dqt-ui:1.2 internal.registry.lan/dqt:1.2
docker push internal.registry.lan/dqt:1.2

# 2. Зеркало PyPI колеса
pip download dqtui==1.2 -d ./wheels

# 3. На целевой машине
pip install --no-index --find-links ./wheels dqtui
```

Никаких внешних запросов на этапе инсталляции и работы.

## 8. Артефакты для регулятора

* HTML-отчёт → файл, прикладывается к проверке.
* Audit log → текстовый JSONL, проверяется `dqt audit verify`.
* Run record (JSON) → `dqt runs show <id>` — машинно-читаемая выписка.
* Регуляторный отчёт → `dqt regulatory render cbr_483p <run_id> -o report.md`.
